Sample Collection Latency

Architectural Perf Core Latency

We know that during context switch, x86_pmu.drain_pebs() or intel_pmu_drain_pebs_icl() in our case, will be called to collect PEBS samples. By measuring how many times it is called, we can calculate the average duration that a group of samples sits in the PEBS buffer.

First run 
clear@clr ~ $ sudo timeout 10s  bpftrace -e 'kfunc:intel_pmu_drain_pebs_icl {@[kstack]=count();}'
